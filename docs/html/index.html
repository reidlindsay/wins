  • wins: WiNS: Wi-reless N-etwork S-imulator.
    • wins.backend: Package implements extension modules for WiNS. Includes all SWIG-wrapped objects and packages for interacting with Hydra PHY.
      • wins.backend.crc: Interface to CRC class from IT++
      • wins.backend.dot11n_backend: Interface to IEEE 802.11n Physical Layer
      • wins.backend.itpp: Interface to helper functions from IT++ library
      • wins.backend.waveform: Wrapper for IT++ Complex Matrices
    • wins.base: Implement common functionality for all wins objects ; contains Base and Reference classes.
    • wins.channel: Package to implement channel.
      • wins.channel.breakpoint: Breakpoint channel model uses freespace propagation until specified break point distance; contains BreakPoint classes.
      • wins.channel.channelbase: Base classes needed for implementing channel to connect nodes together; contains Channel and ChannelModel classes.
      • wins.channel.fixdelay: Wrapper to be used to apply a fixed delay to any ChannelModel by modifying the 'cm-delay' annotation.
      • wins.channel.interface: Interface for connecting nodes to Channel; contains ChannelInterface class.
      • wins.channel.propagation: Basic propagation model for wireless channel; contains Propagation class.
      • wins.channel.radio: Interface for connecting nodes to a wireless channel; contains Radio class.
      • wins.channel.refprop: Reference model for propagation over wireless channel; contains ReferencePropagation class.
    • wins.const: Defines all constants used in wins.
    • wins.crc: Contains class for creating and modifying cyclic-redundancy check (CRC); contains CRC32 class and helper functions.
    • wins.digital: Package for digital communications algorithm and models.
      • wins.digital.modulation: Base class for implementing digital modulation; contains Modulation class.
      • wins.digital.mqam: Model for quadrature amplitude modulation (QAM); contains MQAM class.
      • wins.digital.rcpc: Model for (133,171) rate-compatible punctured convolutional (RCPC) code; contains RCPC class.
      • wins.digital.reedsolomon: Model for Reed-Solomon error-correcting codes; contains ReedSolomon class.
    • wins.element: Basic building blocks for all protocol objects; contains Element and Port classes.
    • wins.fsm: Implementation of finite state machine; contains FSM and Timer.
    • wins.helper: Helper classes and methods for wins.
    • wins.ieee80211: Module for protocols in IEEE 802.11 wireless standard.
      • wins.ieee80211.dcf: Distributed Control Function of IEEE 802.11; contains DCF class.
      • wins.ieee80211.dot11_support: Packet definitions, enumerations, and helper functions for IEEE 802.11 protocol.
      • wins.ieee80211.dot11a: Implements IEEE 802.11a PHY protocol; contains Dot11APHY and Dot11ARadio classes.
      • wins.ieee80211.dot11a_support: Packet definitions, enumerations, and helper functions for IEEE 802.11a physical layer protocol.
      • wins.ieee80211.dot11n: Implements IEEE 802.11n PHY protocol; contains Dot11NPHY and Dot11NRadio classes.
      • wins.ieee80211.dot11n_channel: IEEE 802.11n channel models; contains Dot11NChannel class.
      • wins.ieee80211.dot11n_dsp: Implements baseband digital signal processing for waveform-level simulation of IEEE 802.11n PHY protocol; contains Dot11N_DSP class.
      • wins.ieee80211.dot11n_support: Packet definitions, enumerations, and helper functions for IEEE 802.11n physical layer protocol.
      • wins.ieee80211.navtimer: Timer for Network allocation vector (NAV), used with virtual carrier sense; contains NAVTimer class.
    • wins.mac: Module for protocols in MAC protocols.
      • wins.mac.arf: Auto Rate Fallback (ARF) protocol; contains ARF class.
      • wins.mac.arf_support: Packet definitions, enumerations, and helper functions for ARF protocol.
      • wins.mac.rateadaptation: Rate Adaptation module; contains RateAdapt and RAI classes.
      • wins.mac.rbar: Receiver-Based AutoRate (RBAR) protocol; contains RBAR class.
      • wins.mac.rbar_support: Packet definitions, enumerations, and helper functions for RBAR protocol.
    • wins.mobile: Package for implementing mobile or wireless features.
      • wins.mobile.motion: Base class for simulating motion of Node objects; contains Motion class.
      • wins.mobile.waypoint: Random Waypoint mobility model; contains RandomWaypoint class.
    • wins.net: Module for Network layer protocols.
      • wins.net.dsr: Implementation of Dynamic Source Routing protocol (RFC 4728); contains DSR class.
      • wins.net.dsr_rreq: Implementation of Route Request Table for DSR; contains RouteRequestTable and RouteRequestEntry classes.
      • wins.net.dsr_support: Packet definitions, enumerations, and helper functions for DSR protocol.
      • wins.net.routecache: Implementation of Route Cache for DSR; contains RouteCache class.
    • wins.packet: Contains classes for creating and modifying packets; contains Packet and ANNO.
    • wins.protocol: Package to implement base classes for protocols.
      • wins.protocol.aloha: Implementation of pure Aloha MAC protocol; contains Aloha class.
      • wins.protocol.arp: Implementation of address resolution protocol; contains ARP class.
      • wins.protocol.arp_support: Helper methods and support classes for ARP implementation.
      • wins.protocol.csmac: Base class for implementing carrier sense MAC control protocols such as CSMA/CA; contains CSMAC class.
      • wins.protocol.csphy: Base class for implementing physical layer with carrier sense support which is needed for implementing CSMAC MAC protocols; contains CSPHY class.
      • wins.protocol.mac: Base class for implementing medium-access control (or Layer 2) protocols; contains MAC class.
      • wins.protocol.net: Base class for implementing network (or Layer 3) protocols; contains NET class.
      • wins.protocol.phy: Base class for implementing physical layer (or Layer 1) protocols; contains PHY class.
      • wins.protocol.route_support: Packet definitions, enumerations, and helper functions for Routing protocol.
      • wins.protocol.routetable: Base class for maintaining route table; contains RouteTable class.
      • wins.protocol.routing: Base class for implementing routing protocols; contains Routing class.
    • wins.queue: Implementation of FIFO and priority queue; contains Queue class.
    • wins.trace: Implementation of trace functionality; contains Trace and Traceable.
    • wins.traffic: Module for traffic generating modules.
      • wins.traffic.agent: Agent for generating and receiving traffic; contains AGT and Agent.
    • wins.util: Package for other utilities.
      • wins.util.fileio: Helper methods to read/write data from/to files as dictionaries.
      • wins.util.modpath: Modify current PATH.
      • wins.util.monitor: Monitor progress by printing status messages at regular intervals.
