 44      default ARF operates using CSMA/CA (i.e. not MACAW). To enable RTS/CTS 
 45      messages, set `usecsma` to False. 
 46   
 47      ARF Timeout 
 48      =========== 
 49      The ARF timer is used to indicate that the rate of transmissions should be 
 50      increased prior to having `nsuccess` consecutive ACKs. This mechanism is 
 51      meant to allow ARF to explore higher data rates when traffic is sparse. 
 52   
 53      The timer can be implemented as a time-based or packet-based timer. The 
 54      time-based timer is a straight forward timeout timer that waits for a 
 55      duration before firing and indicating that the rate of the transmission 
 56      should be increased. The packet-based timer uses the "virtual timer" 
 57      approach described by Qiao, Choi, and Shin (2002). This "timer" counts the 
 58      number of transmission attempts made rather than using a time-based timer. 
 59   
 60      This module implements the ARF timer as a time-based timer. 
 61   
 62      :CVariables: 
 63       * `base`: Property to access base rate enumeration from `ra`. 
 64       * `rates`: Property to access ordered list of valid rate enumerations. This 
 65                  is ordered based on the data rates reported by `get_datarate()`. 
 66   
 67      :IVariables: 
 68       * `ra`: `RateAdapt` module. 
 69       * `ackcount`: Dictionary containing number of consecutive successful ACKs 
 70                     received (negative values indicate consecutive dropped ACKs). 
 71                     Values are keyed by destination address. 
 72       * `arftimer`: Dictionary containing ARF timers used for updating transmit 
 73                     data rate. Values are keyed by destination address. This 
 74                     module implements ARF timers as packet-based timers. 
 75       * `probation`: Dictionary containing probation flags indicating if 
 76                      probation period has been entered; values are keyed by 
 77                      destination address. 
 78       * `nsuccess`: Threshold for number of consecutive ACKs that must be 
 79                     received prior to increasing the data rate. 
 80       * `nfailure`: Threshold for number of consecutive ACK failures that 
 81                     will trigger a decrease in the data rate. 
 82       * `timeout`: Timeout value for ARF timer; if exceeded, then increase rate.
