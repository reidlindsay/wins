Package protocol

source code

Package to implement base classes for protocols.

Most protocols should include four ports, namely:

  1. "TXU" - sends received packets to upstream element.
  2. "RXU" - receives traffic from an upstream element.
  3. "TXD" - sends traffic to a downstream element.
  4. "RXD" - receives traffic from a downstream element.

Maintaining this convention will make accessing Port configurations for various protocols more straight forward. Some protocols may strictly enforce all or part of this convention.

Submodules [hide private]
  • wins.protocol.aloha: Implementation of pure Aloha MAC protocol; contains Aloha class.
  • wins.protocol.arp: Implementation of address resolution protocol; contains ARP class.
  • wins.protocol.arp_support: Helper methods and support classes for ARP implementation.
  • wins.protocol.csmac: Base class for implementing carrier sense MAC control protocols such as CSMA/CA; contains CSMAC class.
  • wins.protocol.csphy: Base class for implementing physical layer with carrier sense support which is needed for implementing CSMAC MAC protocols; contains CSPHY class.
  • wins.protocol.mac: Base class for implementing medium-access control (or Layer 2) protocols; contains MAC class.
  • wins.protocol.net: Base class for implementing network (or Layer 3) protocols; contains NET class.
  • wins.protocol.phy: Base class for implementing physical layer (or Layer 1) protocols; contains PHY class.
  • wins.protocol.route_support: Packet definitions, enumerations, and helper functions for Routing protocol.
  • wins.protocol.routetable: Base class for maintaining route table; contains RouteTable class.
  • wins.protocol.routing: Base class for implementing routing protocols; contains Routing class.
